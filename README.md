# mipt-securityprototype
Научно-исследовательский прототип динамической защиты робототехнических устройств на основе детерминированного хаоса.

Репозиторий содержит программную реализацию концепции динамического изменения параметров сетевой инфраструктуры (Moving Target Defense, MTD). Данная технология применяется для защиты каналов управления киберфизических систем (роботов, беспилотных летательных аппаратов, промышленных контроллеров) от сетевого сканирования, анализа трафика и целенаправленного подбора портов.

# 1. Быстрый запуск

Для проверки работы алгоритма, построения графиков и анализа распределения плотности вероятности без локальной настройки окружения можно использовать интерактивную среду Google Colab.

# 2. Архитектура и математическая модель системы

Концепция MTD в данном прототипе реализуется через непрерывную динамическую смену активных сетевых портов. Главная инженерная проблема MTD заключается в синхронизации: если пульт и робот не будут знать текущий порт друг друга, связь прервется. 
Данный прототип решает проблему синхронизации с помощью детерминированного хаоса, что полностью исключает необходимость передавать номера портов по открытой сети.

### 2.1 Генератор последовательностей на основе Tent map
В качестве генератора хаотической динамики используется одномерное палаточное отображение (Tent Map):

* При x < 0.5: x_{n+1} = mu * x_n
* При x >= 0.5: x_{n+1} = mu * (1 - x_n)

Параметр нелинейности mu выбран равным 1.9999. Это значение максимально близко к 2, что обеспечивает режим развитого хаоса, высокую чувствительность к начальным условиям и широкий спектр частот.
Защита от деградации хаоса: любое хаотическое отображение на реальном компьютере из-за ограниченной точности вычислений (формат Float64) со временем зацикливается в нуле или фиксированных точках. В алгоритм встроена система триггеров: если состояние x приближается к критическим границам (0.0, 0.5, 1.0) ближе чем на 10^{-10}, выполняется принудительный сдвиг фазовой траектории на константу 0.123456789 с возвратом в интервал через оператор остатка от деления % 1.0.

### 2.2 Нелинейная трансформация плотности вероятности
Классическое палаточное отображение генерирует числа с равномерной плотностью вероятности на интервале (0, 1). Если злоумышленник перехватит часть трафика, он сможет построить статистическую модель равномерного распределения и оптимизировать стратегию сканирования.
Для усложнения анализа введена синусоидальная трансформация плотности:
y_n = sin(pi * x_n)
В результате равномерное распределение трансформируется в нелинейное U-образное распределение. Частота выбора портов искусственно смещается к краям выделенного диапазона. 
Злоумышленник сталкивается со статистической аномалией: порты меняются хаотически во времени, но их итоговая гистограмма частот имеет глубокий провал в центре. Это ломает стандартные алгоритмы статистического предсказания случайных чисел, но сохраняет полную синхронность между легитимным пультом и роботом.

### 2.3 Масштабирование в пространство портов
Трансформированное хаотическое значение y_n, лежащее в диапазоне (0, 1), линейно масштабируется в целевой рабочий диапазон портов и приводится к целочисленному типу int:
Port_n = int(Port_min + y_n * (Port_max - Port_min))
В прототипе используется выделенный диапазон от 10000 до 11000, обеспечивающий емкость пространства в 1000 возможных состояний.

### 2.4 Обеспечение отказоустойчивости связи
Для компенсации сетевых задержек и возможной микро-десинхронизации часов пульта и робота, на стороне робота реализовано скользящее окно избыточности. Робот одновременно держит открытыми 3 порта:
1. Прошлый порт (на случай задержки пакета в пути).
2. Текущий порт (расчетная точка связи).
3. Следующий порт (превентивное открытие для упреждения десинхронизации).
Пульт управления отправляет пакет строго на текущий расчетный порт. Связь гарантирована в 100% случаев.

# 3. Модель нарушителя и оценка эффективности
### 3.1 В рамках симуляции (цикл из 1000 шагов) моделируется поведение атакующего со следующими характеристиками:
 * Атакующий знает рабочий диапазон портов (10000–11000).
 * Атакующий осуществляет слепое высокоскоростное сканирование, проверяя 3 случайных порта на каждом шаге времени.
 * Атакующий не имеет доступа к начальному секретному ключу, разделяемому между пультом и роботом.
### 3.2 Метрики эффективности по результатам тестов:
 * Доступность канала связи: 100.00% (легитимный пульт всегда попадает в открытый порт робота благодаря синхронному генератору).
 * Вероятность сетевого пробития: менее 0.9% (математическое ожидание случайного попадания 3 портов сканера в 3 открытых порта робота из 1000 доступных). Динамическая смена адреса полностью нивелирует преимущество высокой скорости сканирования.

# 4. Разбор структуры исходного кода

 * tent_value(x) - Вычисляет следующую итерацию палаточного отображения, проверяет условия вырождения хаоса и корректирует траекторию.
 * transf(x) - Модуль искажения статистики. Применяет тригонометрическую функцию для изменения плотности распределения с равномерной на U-образную.
 * map_value(x) — Инженерный интерфейс. Переводит абстрактное хаотическое число в реальный номер порта в диапазоне 10000–11000.
 * np.random.seed(...) - Инициализация генератора случайных чисел. Использует системное время в миллисекундах для генерации уникального начального ключа.
 * x_1, x_2, x_2pro - Переменные состояния. Обеспечивают независимый параллельный расчет хаоса на пульте и роботе, а также хранят историю для скользящего окна.
 * any(hport in ports for hport in hs) - Логика проверки пробития. Определяет, смог ли сканер хакера угадать хотя бы один из трех активных портов робота.
 * Блок matplotlib - Скрипт визуализации. Формирует три графика: временную шкалу изменений портов, сравнительную гистограмму распределения плотности вероятности и итоговую диаграмму защищенности системы.

# 5. Ограничения текущей модели и направления исследований
Данный прототип создавался для проверки концепции, поэтому содержит ряд упрощений. Перевод системы в класс промышленного ПО требует глубокой междисциплинарной проработки:
* **Деградация хаоса на реальном железе:** Вычисления в формате `Float64` имеют конечную точность. Реализация на микроконтроллерах (STM32, ESP32) или ПЛИС (FPGA) потребует перевода формул отображения Тента в целочисленную арифметику с фиксированной запятой.
* **Проблема первичного рукопожатия:** В симуляции пульт и робот синхронизируются мгновенно через единую переменную. В реальных условиях необходим защищенный протокол первоначального обмена фазовым состоянием генераторов.
* **Помехоустойчивость и дрейф времени:** При глубоком затухании радиосигнала окно из 3 портов может быть исчерпано. Требуется разработка алгоритмов адаптивного расширения окна и протоколов высокоточной временной синхронизации (типа PTP) для компенсации дрейфа аппаратных кварцевых резонаторов.
* **Многомерный криптоанализ:** Одномерные хаотические отображения уязвимы к методам восстановления фазового пространства с помощью машинного обучения. Перспективным направлением является переход к многомерным гиперхаотическим системам (например, аттракторам Лоренца или Рёсслера).
